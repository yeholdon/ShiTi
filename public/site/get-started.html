Quick Start

第一次接手 ShiTi，从哪里开始最省时间。

这里是管理台和 OpenAPI 之间的过渡层，帮助新同事先建立最基本的操作路径。

1. 起基础设施

确保 Postgres、Redis、MinIO 可用，再启动 API。健康状态正常后再进入后台。

2. 进入后台

在 `/admin` 注册、创建租户、加入租户，然后先建立 taxonomy 和标签，再建题目与文档。

3. 查接口细节

如果要对接业务系统或脚本，用 `/docs` 和 `/docs/openapi.json` 直接查字段和接口形态。

Takeover Checklist

接手项目时最短的一条路径

先跑 `npm run dev:up`

它会把基础设施、Prisma、seed 和 API 一次拉起，比手工逐条执行更不容易漏步骤。

先看 `/` 再进 `/admin`

公开前台负责建立上下文，后台负责执行动作，两个入口职责不同。

把 `/docs` 当作事实源

脚本接入、联调和字段核对都应优先参考 Swagger 和 OpenAPI JSON。

把 `TODO.md` 当作进度源

仓库当前完成度和后续拓展入口优先看 TODO，而不是回头翻聊天记录。
