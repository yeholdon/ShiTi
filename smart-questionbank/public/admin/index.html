Service Gates

健康检查含义

`API 状态` 对应 liveness；`Readiness` 会继续探测数据库、Redis 和 MinIO，可用时才适合执行写入、导出和资源流转。

Primary Surfaces

管理面职责

上半部分处理高频操作，下半部分负责深度编辑与审计。你可以把它当作一个面向租户的后端 runbook，而不是 CMS 首页。

What To Watch

排障优先级

先看健康状态，再看审计和活动流；若是导出或资源问题，优先确认租户上下文、owner/admin 权限和对象是否被正确选中。

Console Runbook

这页到底该怎么用

1. 先确认服务

先看 `API 状态` 和 `Readiness`。如果 readiness 失败，不要急着导出或上传资源，通常是数据库、Redis 或 MinIO 没准备好。

2. 再确认上下文

运维视角里，最常见的人为错误不是接口坏了，而是“操作到了错误租户”或“没选中文档 / 题目 / 布局对象”。顶部指挥区就是为此存在。

3. 最后下钻日志

审计日志看系统事实，活动日志看本次交互轨迹。先看聚合分布，再看明细，最后结合下方工作区复现操作。

Question Operations

题库工作区

创建、筛选和选中题目，再进入下方的深度编辑区。

Questions

新建题目

这里负责题目入口和列表检索。你可以先创建空题，再通过筛选器收敛到特定题型、标签、教材、章节和可见性范围，然后进入下方详情区编辑。

操作说明

创建会走 `POST /questions`；列表会拉 `GET /questions`，并带上 `tags, summary` 以便快速判断题目内容和 taxonomy 绑定情况。

POST /questions GET /questions admin / owner 写入 member 只读

Documents

创建文档

这里是文档主入口。`handout` 适合讲义和版式元素混排，`paper` 更偏试卷。创建后可进入下方文档编排区做细化操作。

操作说明

创建文档后，文档列表会展示当前项数、题目数、布局项数和最近一次导出状态，便于快速判断这份文档是否已经成型。

POST /documents GET /documents 文档摘要 / 最近导出

Layout

创建布局元素

布局元素只服务于 `handout`，适合插入导语、节间说明、提示语或版式占位文本。它们会在导出链路里与题目一起渲染。

POST /layout-elements GET /layout-elements 仅 handout 使用

Assets

上传资源

资源上传会先生成元数据和预签名地址，再由浏览器把文件传给对象存储。后台会保留原始文件名，并阻止删除仍被引用的资源。

POST /assets/upload GET /assets DELETE /assets/:id 引用保护
