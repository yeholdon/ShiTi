ShiTi API Docs

拾题后端接口说明

这是当前仓库内置的 API 文档入口。当前 `/docs` 由 Swagger UI 提供，这个静态页主要补充运行约束、角色模型和运维提示。

Auth & Tenant

认证与租户

POST/auth/register
POST/auth/login
POST/tenants
GET/tenants/resolve
POST/tenant-members

Tenant-scoped requests usually need `Authorization: Bearer ...` and `X-Tenant-Code`.

Roles

角色权限

READmember / admin / owner
WRITEadmin / owner

当前题目、文档、taxonomy、标签、资源、布局元素以及导出任务的管理类写接口要求 `admin` 或 `owner`。普通 `member` 仍可读取租户内资源，但不能自助把自己提升成更高角色。

Health

健康检查

GET/health
GET/health/ready

`/health` 用于 liveness，`/health/ready` 会检查 Postgres、Redis、MinIO。

Metrics

运行指标

GET/metrics

返回 Prometheus 风格文本指标，当前包含 API uptime、HTTP 请求总数、按状态码聚合和按 method/status 聚合。

Errors

错误响应

JSON{ statusCode, message, error, path, timestamp, requestId }

4xx/5xx 现在统一返回错误包。`error.code` 适合前端做稳定判断，`message` 保留人类可读信息，`requestId` 可用于和服务端日志关联。

Observability

请求追踪

HTTPX-Request-Id
LOG{ type, requestId, method, path, statusCode, durationMs, tenantCode, tenantId }

所有 API 响应都会带 `X-Request-Id`，也支持客户端自带该 header。服务端会输出结构化 JSON 请求日志，便于排查错误和关联租户流量。

Audit

审计日志

GET/audit-logs
GET/audit-logs/stats

审计日志持久化存储在 Postgres，按租户隔离。`/audit-logs` 支持按 `action`、`targetType`、`userId`、`since`、`until` 和 `limit` 过滤，`/audit-logs/stats` 返回动作、用户和目标类型聚合；这两个接口当前要求 `admin` 或 `owner`。
