Operator Guidance

看到这些状态后该怎么判断

Liveness 失败

说明 API 进程本身已经不可用，优先检查生命周期日志和进程退出原因。

Readiness 失败

说明 API 还活着，但数据库、Redis 或 MinIO 至少有一项不满足操作条件。

Metrics 失败

说明观测面已有盲区，不一定代表业务不可用，但排障成本会明显上升。

依赖出现告警

先判断导出、上传、清理和审计这些依赖敏感动作是否需要暂停。
